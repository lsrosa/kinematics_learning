\section{Results}\label{sec: results}

\subsection{Testing Environment}\label{subsec: test env}

We build upon the \texttt{Reacher-v4} environment from OpenAI Gym \cite{gym}, which consists of a 2D serial manipulator with $2$ joints simulated using MuJoCo \cite{todorov2012mujoco}.
%
In order to evaluate the proposed approach for more realistic robots, we extend the environment enabling for 3D, and adding models for robots up to $7$ joints\footnote{Publicly available source code: \href{https://github.com/lsrosa/kinematics\_learning}{https://github.com/lsrosa/kinematics\_learning}}.

The $\nn$ modules are implemented using PyTorch \cite{pytorch}, allowing for integration with OpenAI Gym, and the derivatives used in Section \ref{subsec: loss and acc} are computed using PyTorch's Autograd \cite{paszke2017automatic}.

\subsection{Parameters Tuning}\label{subsec: tuning}

%
The hyper-parameters search space presented in Table \ref{tab: hyperparams search space} for robots $\nDims=\{2, 3\}$ and $\nJoints=\{2,\ldots,7\}$
\footnote{We use an Intel i9 computer with 16GB of RAM and without GPU.}.

Table \ref{tab: tune fkine} presents the hyper-parameters found for $\nnMono$ and $\nnLink$ for robots operating in 2D and 3D, and with $\nJoints=\{2,\ldots, 7\}$. 
%
The found hyper-parameters values demonstrate the consequences of using a monolithic architecture: the complexity of the function being approximated grows with $\nJoints$ (as described in Equation \ref{eq: x estimate in function of q}), and hence a larger \gls{nn} is more suitable.
%
In comparison, the function being approximated with the proposed architecture is constant in $\nJoints$, and increasing $\nJoints$ increases the number of approximators with constant complexity.

As result, the proposed architecture is expected to scale better with $\nJoints$ than its monolithic counterpart, both in precision and sample efficiency.

Figure \ref{fig: loss fkines} presents the losses when learning $\nnMono$ and $\nnLink$, which shows that the proposed architecture outperforms the monolithic approach found used in the literature.

\begin{figure*}

	\caption{Joints position estimation comparison for $\nnMono$ and $\nnLink$.}

Next, to demonstrate the accuracy impacts, we estimate the end-effector velocity $\dot{\pos}_j = \left[\jacob\dot{\rotAngle{}}\right]_j$, where $\jacob$ is the unknown Jacobian matrix of the robot, as $\prediction{\dot{\pos}}_j = \left[\frac{\partial\nn}{\partial\rotAngle{}}\dot{\rotAngle{}}\right]_j$.
%
That is, we use the learned network for estimating the unknown Jacobian of the model, which has its well-known applications in inverse kinematics \cite{waldron1985study} computation, Cartesian impedance controllers \cite{sanchez2010cartesian}, and tracking controllers \cite{chen2017tracking}.
%
Figure \ref{fig: ee vel} shows the velocity estimation and ground-truth obtained from the simulator for $\nDims=3$ and $\nJoints=7$.
%
The better accuracy of the proposed architecture indicates its adequacy for being used in higher level applications, while the monolithic architecture poor accuracy might impact higher level applications badly.
